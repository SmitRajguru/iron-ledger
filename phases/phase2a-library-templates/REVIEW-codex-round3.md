1. **A1 routine projection honors voids — FIXED:** `client/src/lib/library.js` `project()` builds a global `voidedIds` set, skips any voided event for both exercises/routines, applies tombstones via `payload.deleted` (`delete` from map), and under full rebuild (`mergeOrdered` → `project`) a voided routine event cannot re-win; voiding a non-existent id is a no-op, and chains like `v1<-v2<-v3` correctly leave only `v3`.  
2. **HIGH lost-update race — NOT-FIXED:** `client/src/Routines.svelte` chain serialization is improved, but `chainHead` is cleared as soon as the promise chain settles (not when projection rebuild completes), so a next mutation can fall back to stale `get(routinesMap)` in a narrow post-queue/pre-rebuild window and still drop a prior edit.  
3. **MED `_enq` fallback — NOT-FIXED:** `client/src/lib/sync.js` does keep a module-level monotonic counter and strips `_enq` before POST, but seeding is still racy because `startSync()` calls `seedEnq()` without awaiting it while `queue()` can early-return from `seedEnq()` once `enqSeeded` flips, allowing first enqueue before true `max(persisted, outbox)` initialization completes.  
**NEW bug:** in `Routines.svelte` `mutateRoutine`/`createRoutine`, if one `queue()` rejects, `mutateChain` stays rejected (no recovery catch/reset), so subsequent routine edits won’t execute and `chainHead` cleanup can be skipped.
